# Skill: View Rankings — Show a Question Page to the Player

## Trigger
Player asks to see a question, its top answers, or its rankings.

## Step 1: Fetch the Question Page

Fetch the question page (auto-generated):

`https://raw.githubusercontent.com/context-game/context-game/main/wiki/qa/<slug>/index.md`

This file is auto-generated by the reindex script and contains:
- Question title and stats (entries, judgments, participants)
- Top 10 entries with Elo, W/L record, author info
- Pending entries (1-4 comparisons — needs more judgments)
- Unjudged entries (0 comparisons)
- Player location aggregation
- Poll answers

For programmatic access via a private repo, use the GitHub API:
`GET https://api.github.com/repos/context-game/context-game/contents/wiki/qa/<slug>/index.md`

## Step 2: Display the Question

Show the player something like:

```
## Question: [Title]
[N] entries · [M] judgments · [P] participants
Phase: [collecting/judging]
[Location tag, if any]

### Top 10
1. [Entry title] — Elo [score] — [W-L record] — by [author or anonymous]
2. [Entry title] — Elo [score] — [W-L record] — by [author or anonymous]
...

### Pending (needs more judgments)
- [Entry title] — [X] of 5 comparisons — Elo [score]

### Unjudged
- [Entry title]

[Location aggregation, if any]
[Poll/join answers, if any]
```

## Step 3: Offer Actions

After showing the question, ask:

"Want to:
1. **Write your own entry** — answer this question
2. **Judge** — compare two entries blind
3. **Read the full entry** — see a specific answer in detail
4. **See player profiles** — check out who's playing"

## Step 4: Read a Specific Entry in Detail

Player wants to read entry #N. Fetch the file from:

`https://raw.githubusercontent.com/context-game/context-game/main/wiki/qa/<slug>/entries/<filename>`

Show the full content with frontmatter visible (it's already public in git).

## Step 5: See Player Profile

Player wants to check out an author. Fetch:

`https://raw.githubusercontent.com/context-game/context-game/main/wiki/players/<author>.md`

Show: display_name (if set), links (if set), location (if set), entries count, top-10 finishes, judgments given.

## Edge Cases

| Situation | What to do |
|---|---|
| Question has 0 entries | Show: "Nobody has answered yet. Be the first!" Offer to submit an entry. |
| Question has 0 judges | Show: "These entries need judgments to rank. Can you judge?" |
| Entry has fewer than 5 comparisons | Show it in "Pending" section. Mark "needs X more comparisons." |
| Question page doesn't exist yet (newly created, reindex hasn't run) | Fetch `index.md` instead, or wait for the PR to merge. |
